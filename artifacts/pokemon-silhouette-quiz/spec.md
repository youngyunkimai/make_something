## Overview
"누구의 그 포켓몬?" — 1세대 포켓몬 151종의 실루엣을 보고 4지선다로 이름을 맞추는 퀴즈 게임. 5문제 단위 라운드, 3단계 힌트, 연속 정답 보너스로 구성된다.

## 시나리오

### 1. 퀴즈 시작
[상황] 사용자가 메인 화면에서 '시작' 버튼을 누르면
[동작] 시스템이 1세대 포켓몬 중 랜덤 5종을 선택하고, 첫 번째 문제의 실루엣과 4개 보기를 표시한다

성공 기준:
- [ ] '시작' 버튼 클릭 → 실루엣 이미지 1개 + 보기 버튼 4개 표시
- [ ] 보기 4개 중 정답 1개 포함, 나머지 3개는 오답 (1세대 포켓몬 중 랜덤)
- [ ] 보기에 한국어 + 영어 병기 표시 (예: "피카츄 (Pikachu)")
- [ ] 문제 번호 "1 / 5" 표시

### 2. 정답 선택
[상황] 실루엣 문제가 표시된 상태에서 사용자가 정답 보기를 클릭하면
[동작] 시스템이 실루엣을 원본 이미지로 전환하고, "정답!" 메시지와 점수를 표시한 뒤 다음 문제로 넘어간다

성공 기준:
- [ ] 정답 클릭 → 실루엣이 컬러 이미지로 전환
- [ ] "정답!" 텍스트 표시
- [ ] 기본 점수 100점 획득, 점수 표시 갱신
- [ ] 연속 정답 시 콤보 표시 (예: "2 콤보!")
- [ ] 결과 확인 후 다음 문제로 전환되며 문제 번호가 갱신된다 (예: "1 / 5" → "2 / 5")

### 3. 오답 선택
[상황] 실루엣 문제가 표시된 상태에서 사용자가 오답 보기를 클릭하면
[동작] 시스템이 정답을 공개하고, 콤보를 초기화하고, 다음 문제로 넘어간다

성공 기준:
- [ ] 오답 클릭 → 선택한 보기에 오답 표시, 정답 보기 강조
- [ ] 실루엣이 컬러 이미지로 전환 (정답 포켓몬 공개)
- [ ] 점수 변동 없음 (0점)
- [ ] 콤보 텍스트가 화면에서 사라진다 (콤보 초기화)
- [ ] 결과 확인 후 다음 문제로 전환되며 문제 번호가 갱신된다

### 4. 힌트 사용
[상황] 실루엣 문제가 표시된 상태에서 사용자가 '힌트' 버튼을 누르면
[동작] 시스템이 단계별 힌트를 공개하고 획득 가능 점수를 감소시킨다

성공 기준:
- [ ] 힌트 1회 클릭 → 타입 정보 표시 (예: "전기 타입"), 획득 가능 점수 70점으로 감소
- [ ] 힌트 2회 클릭 → 도감번호 표시 (예: "도감번호: #025"), 획득 가능 점수 40점으로 감소
- [ ] 힌트 3회 클릭 → 실루엣 밝기 50%로 변경 (반쯤 공개), 획득 가능 점수 20점으로 감소
- [ ] 3회 사용 후 힌트 버튼 비활성화

### 5. 결과 요약
[상황] 5문제를 모두 풀면
[동작] 시스템이 결과 요약 화면을 표시한다

성공 기준:
- [ ] 총점 표시 (예: "총점: 380점")
- [ ] 정답률 표시 (예: "정답률: 4 / 5")
- [ ] 최대 콤보 표시 (예: "최대 콤보: 3")
- [ ] 틀린 포켓몬 목록 표시 (실루엣 → 정답 이미지 + 이름)
- [ ] '다시 하기' 버튼 표시
- [ ] '다시 하기' 클릭 → 문제 번호 "1 / 5", 점수 "0점"으로 초기화되고 새 실루엣과 보기 4개 표시

## 범위

### 포함
- 메인 화면 (시작 버튼)
- 퀴즈 화면 (실루엣 + 4지선다 + 힌트 + 점수)
- 결과 화면 (요약 + 다시 하기)
- PokeAPI 연동 (1세대 포켓몬 데이터 + 스프라이트)
- 한국어/영어 병기 포켓몬 이름

### 제외
- 세대 선택 기능 — 1세대 고정으로 스코프를 줄이기 위해
- 리더보드/랭킹 — 백엔드 필요, 5시간 범위 초과
- 사용자 인증 — 과제 제출용으로 불필요
- 난이도 선택 — 단일 난이도로 완성도 집중
- 사운드 효과 — 구현 우선순위 낮음

## 전제 조건
- PokeAPI (https://pokeapi.co/)가 정상 동작하며, 1세대 포켓몬의 한국어 이름을 species 엔드포인트에서 제공한다
- 포켓몬 스프라이트 이미지가 official-artwork 형식으로 제공된다
- CSS `filter: brightness(0)`로 실루엣 효과를 구현할 수 있다
